# 日语阅读助手

一个手机和网页可用的日语阅读辅助工具原型。

## 在线页面

GitHub Pages 启用后，页面地址会是：

https://1274213542.github.io/Rachel7802/

说明：GitHub Pages 只能托管静态网页，不能直接运行 `server.py` 后端。因此在线页面可以打开和使用阅读、假名、收藏、标记等前端功能；更准确的联网查词、中文释义和例句接口需要后续把 `server.py` 单独部署到后端平台。

## 功能

- 输入或粘贴日语文本
- 自动标记包含汉字的词语
- 桌面端悬停显示读音和简短中文释义
- 桌面端点击词语打开页面内详情面板
- 手机端轻触显示或隐藏注释，并可进入详情
- 一键显示全文假名
- 单独保存某个词的假名显示
- 给句子添加浅色荧光重点标记
- 阅读区编辑文字
- 收藏单词和整篇文本记录
- 收藏夹保存在当前浏览器本地

## 本地使用

推荐通过本地服务打开页面，才能使用在线查词与例句查询。

```bash
python3 server.py --host 0.0.0.0 --port 8787
```

如果要在手机上打开，需要在电脑上启动本地服务并使用电脑的局域网 IP，例如 `http://192.168.10.106:8787/`。手机和电脑需要连接同一个 Wi-Fi，不能使用 `127.0.0.1`，因为它在手机上表示手机自己。

## 数据策略

当前版本只显示可追溯来源：

- Jisho / JMdict：用于读音、词性和日英参考释义；不冒充中文释义。
- Tatoeba：用于真实例句；只有查到中文译文时才显示。
- OpenCC：仅用于把 Tatoeba 返回的繁体中文译文转成简体显示，不参与翻译。
- OpenAI API：可选。设置 `OPENAI_API_KEY` 后，会生成 AI 辅助日中释义，并在界面中明确标注为 AI 辅助。

如果没有可靠日中词典释义或中文例句，页面会明确显示“暂无可靠中文释义”或“暂无可靠中文例句”，不会生成模板例句，也不会把英文释义机器翻译后伪装成中文词典。

启动 AI 辅助释义示例：

```bash
OPENAI_API_KEY=你的密钥 python3 server.py --host 0.0.0.0 --port 8787
```
